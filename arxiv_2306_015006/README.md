# DNABERT-2 — Reproduction

**Paper:** [DNABERT-2: Efficient Foundation Model and Benchmark For Multi-Species Genomes](https://arxiv.org/abs/2306.15006)
**Authors:** Zhihan Zhou, Yanrong Ji, Weijian Li, Pratik Dutta, Ramana V Davuluri, Han Liu (Northwestern, Stony Brook) — ICLR 2024

> *Generated by ArXivist from SIR `arxiv_2306_015006`. Independent reproduction, not the authors' official code.*

## What this paper does (plain English)

DNABERT-2 is a genome foundation model that replaces **k-mer tokenization** with **Byte Pair
Encoding (BPE)** — fixing the information-leakage and sample-inefficiency problems of k-mers — and
uses **Attention with Linear Biases (ALiBi)** to handle arbitrarily long DNA sequences plus
**GEGLU** activations and Flash Attention for efficiency. At ~117M parameters it matches the 2.5B
Nucleotide Transformer with ~21× fewer parameters and ~92× less pre-training compute. The paper also
introduces **GUE**, a 28-dataset / 7-task multi-species benchmark.

This repo **fine-tunes the official pretrained DNABERT-2** (`zhihan1996/DNABERT-2-117M` from
HuggingFace — BPE tokenizer + ALiBi encoder bundled) on any GUE task, then evaluates with the paper's
MCC/F1 protocol.

## Fine-tuning recipe (paper Appendix A.3 — high confidence)

AdamW · **lr 3e-5** · β=(0.9, 0.98) · **weight_decay 0.01** · **batch 32** · **warmup 50 steps** ·
per-task epochs (Table 7) · validate every 200 steps, keep lowest-val-loss checkpoint · 3 seeds
averaged in the paper.

## Compatibility notes (DNABERT-2's 2023 code on a modern stack)

DNABERT-2's remote modeling code predates current library versions. The loader
([`models/classifier.py`](src/dnabert2/models/classifier.py)) handles all three known breakages
automatically — **no version pinning or runtime restart needed**:

| Symptom | Cause | Fix applied |
|---|---|---|
| `AttributeError: 'BertConfig' object has no attribute 'pad_token_id'` | their `config.json` omits it; transformers ≥4.50 dropped the default | inject the true value read from their tokenizer (`[PAD]`=3) |
| `RuntimeError: Tensor on device meta is not on the expected device cpu` | `rebuild_alibi_tensor()` builds eager tensors vs meta-device lazy init | `low_cpu_mem_usage=False` |
| `TypeError: dot() got an unexpected keyword argument 'trans_b'` | their `flash_attn_triton.py` uses a Triton 2.x API removed in Triton 3.x | set `attention_probs_dropout_prob > 0`, selecting **the authors' own PyTorch attention branch** (same maths; they intend dropout for fine-tuning) |

Control the last one via `model.attention_dropout` in the config (default `0.1`).

**GPU:** use a GPU runtime for reasonable speed. The Triton kernel is deliberately bypassed, so
`triton` is not required at forward time.

## Reproducibility notes

- **Model** loads directly via `AutoModel` (DNABERT-2 has a proper `model_type` — no vendoring needed,
  unlike some genomic models). BPE tokenizer comes from the same repo, so token ids match the weights.
- **Pooling** for the classification head is **masked mean-pool** (`# ASSUMED`, SIR conf 0.75; the
  paper doesn't specify — `pool: cls` is available in config).
- **Layer count / d_model** are taken from the loaded pretrained config, not hardcoded.
- Single-seed runs here will vary slightly from the paper's 3-seed average.
