## What this paper does

The paper compares three ways of generating Monte Carlo scenarios for financial and
macroeconomic risk factors (equity index, interest rates, inflation, and the full yield
curve): a purely nonparametric **Stationary Bootstrap** that resamples historical blocks of
returns/differences; a semiparametric **VAR-Bootstrap** that fits a VAR(1) model and resamples
its residuals, introducing mean-reversion; and a **Nelson-Siegel VAR-Bootstrap** that applies
the same VAR(1)+residual-bootstrap idea to three latent Nelson-Siegel yield-curve factors
(level, slope, curvature) instead of the raw curve nodes. The empirical results show a
trade-off: the Stationary Bootstrap best preserves historical marginal distributions (lowest
KL divergence), while the two semiparametric methods produce more economically plausible,
mean-reverting, and (for the yield curve) arbitrage-consistent individual trajectories.

## Quick start

```bash
git clone <this-repo> && cd arxivist_bootstrap
pip install -r requirements.txt && pip install -e .
# Supply your own data per data/README_data.md, then:
bash data/download.sh                                  # verifies data is present
python train.py --config configs/config.yaml            # fits all three methodologies
python evaluate.py --config configs/config.yaml --method ns_var_bootstrap
python scripts/plot_figures.py --simulations results/simulations.npz --out-dir results/figures
```

## Usage

### Fit all three methodologies
```bash
python train.py --config configs/config.yaml --freq daily
# or, equivalently:
python scripts/fit_models.py --config configs/config.yaml --freq daily --out-dir results/fitted_models
```
Use `--debug` for a quick smoke test on a truncated window, `--dry-run` to validate the setup
without fitting anything.

### Simulate scenarios
```bash
python scripts/simulate.py --config configs/config.yaml --method ns_var_bootstrap \
    --n-paths 10000 --horizon-years 5 --out results/simulations.npz
```

### Evaluate (reproduce Tables 1-11)
```bash
python evaluate.py --config configs/config.yaml --method ns_var_bootstrap
# writes results/metrics_report.md
```

### Reproduce figures
```bash
python scripts/plot_figures.py --simulations results/simulations.npz --out-dir results/figures
```

### Single-scenario inference
```bash
python inference.py --config configs/config.yaml --method var_bootstrap --horizon 60
```

## Implementation assumptions & reproducibility notes

The paper omits several numeric details needed for an exact reproduction. All are exposed as
explicit, documented config parameters (never silently hardcoded) — see
`sir-registry/arxiv_2606_11859/sir.json` → `implementation_assumptions` / `ambiguities` for
full detail:

| Assumption | SIR confidence | Config knob |
|---|---|---|
| Stationary Bootstrap mean block length (not reported) | 0.4 | `model.stationary_bootstrap.mean_block_length` |
| Nelson-Siegel lambda calibration routine/value (not reported) | 0.6 | `model.nelson_siegel.lambda_calibration` |
| KL-divergence KDE bandwidth (paper flags as consequential but unspecified) | 0.4 | `evaluation.kl_kde_bandwidth` |
| i.i.d. vs. block VAR-Bootstrap residual resampling | 0.65 | `model.var_bootstrap.residual_resampling` |
| Historical estimation sample window (start/end dates not given) | 0.4 | `data.sample_window` |
| VAR(1) estimator (OLS assumed — standard practice, not stated explicitly) | 0.75 | `training.var_estimator` |

**Known deviations**: because the underlying market data is proprietary, this repo cannot be
run out-of-the-box to reproduce the paper's exact numeric tables — you must supply equivalent
data (see `data/README_data.md`). All statistical machinery (transforms, VAR(1) estimation,
Nelson-Siegel fitting, bootstrap simulation, evaluation metrics, arbitrage checks) is
implemented directly from the paper's equations (Sections 2-6, Appendices A-B).
