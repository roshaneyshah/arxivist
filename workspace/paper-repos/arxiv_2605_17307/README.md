A from-scratch PyTorch reproduction of the **SAC-based portfolio allocation** framework from the paper, supporting all five published configurations:

| Config       | Encoder     | Policy                | Reward             | Cash?       | top-k    |
|--------------|-------------|-----------------------|--------------------|-------------|----------|
| `LSTM_1`     | LSTM        | Flat Dirichlet        | Log-return         | yes (flex.) | 20, 30   |
| `LSTM_2`     | LSTM        | Hierarchical Dirichlet| Log-return         | yes (flex.) | 20, 30   |

Evaluated on **NASDAQ-100**, **Nikkei 225** and **Euro Stoxx 50** via **16-fold non-anchored walk-forward** (2009-04 → 2026-03) with adaptive retraining.

## ⚠️ Important caveats

1. **Index membership requires Bloomberg Terminal**, which is paywalled.
   This repo ships a stub that returns **current constituents only** → survivorship-biased reproduction. To remove the bias, drop CSVs into `data/membership/{ndx,nky,sx5e}/` following the format documented there.
2. **Full reproduction is expensive**: ≈14–23 h **per fold** on NVIDIA L4, × 16 folds × 5 configs × 3 markets ≈ **3 400+ GPU-h**.
   Use `--quick-test` to run 1 fold × 2 epochs on a CPU in minutes.
3. Several implementation details are **inferred** (see `# ASSUMED:` comments and the [ambiguities section of the SIR](../../sir-registry/arxiv_2605_17307/sir.json)).

## Quick start

```bash
# 1. Install
pip install -r requirements.txt

# 2. Download price data (yfinance)
bash data/download.sh

# 3. Smoke test (CPU OK)
python train.py --config configs/lstm_2_ndx.yaml --quick-test

# 4. Full training (GPU strongly recommended)
python train.py --config configs/lstm_2_ndx.yaml --seed 42

# 5. Evaluate
python evaluate.py --run-dir runs/lstm_2_ndx_seed42 --benchmark QQQ
```

Or via Docker:

```bash
docker compose -f docker/docker-compose.yml run --rm trainer \
    python train.py --config configs/lstm_2_ndx.yaml --quick-test
```
