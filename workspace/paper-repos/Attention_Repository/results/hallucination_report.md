## Category 2 — Parametric Hallucinations

### P1 — 3-Way Weight Tying
**Severity**: Significant | **SIR Confidence**: 0.82

**Paper says** (Section 3.4):
> "we share the same weight matrix between the two embedding layers and the
> pre-softmax linear transformation"

**Implemented as**:
```python
# transformer.py — tie_weights()
self.tgt_embedding.embedding.weight = self.src_embedding.embedding.weight
self.output_projection.weight = self.src_embedding.embedding.weight
```

**Issue**: "two embedding layers" could mean encoder + decoder embeddings (both tied
to the output projection — 3-way), or only decoder input + output projection (2-way,
with the encoder having a separate embedding matrix).

**Evidence for current choice**: The original tensor2tensor implementation uses 3-way
tying when `shared_embedding=True`, which matches the EN-DE setup (shared vocab).

**Impact if wrong**: +19M parameters (37k × 512) for a separate encoder embedding.
Expected BLEU delta: ±0.1–0.3. Verified in this run: weight tying confirmed active ✓

### P2 — Attention Dropout
**Severity**: Minor | **SIR Confidence**: 0.78

**Paper says** (Section 5.4):
> "We apply dropout to the output of each sub-layer [...] In addition, we apply
> dropout to the sums of the embeddings and the positional encodings."

**Implemented as**: Dropout also applied to attention weights after softmax
(inside `ScaledDotProductAttention.forward()`), at the same rate P_drop=0.1.

**Issue**: The two locations named in Section 5.4 are residual path and embedding
path. Attention weight dropout is not mentioned explicitly.

**Impact**: Minor regularisation difference. Most published reproductions (fairseq,
annotated-transformer) apply attention dropout at the same rate. Expected BLEU
delta: ±0.0–0.2.

### P4 — Positional Encoding Addition Ordering
**Severity**: Minor | **SIR Confidence**: 0.88

**Paper says** (Section 5.4):
> "we apply dropout to the sums of the embeddings and the positional encodings"

**Implemented as**:
```python
# embeddings.py — PositionalEncoding.forward()
x = x + self.pe[:, :x.size(1), :]   # add PE to embedding
return self.dropout(x)               # dropout applied after
```

**Assessment**: This matches the most natural reading of "dropout to the sums" —
dropout is applied to the sum of embedding + PE, not to the embedding alone.
Rated Minor only because the SIR flagged it as an ambiguity; the implementation
is almost certainly correct.

**Impact**: Effectively zero.
