## What This Repo Does

Reproduces the full QSVM + Quantum-SMOTE pipeline from the paper:

1. **Quantum-SMOTE** (Section II) — generates synthetic fraud samples using K-means clustering, amplitude encoding, quantum swap test, and Ry-rotation synthesis
2. **QSVM** (Section III) — maps data to 2ⁿ-dimensional Hilbert space via ZZFeatureMap, computes quantum kernel K(xᵢ, xⱼ) = |⟨φ(xᵢ)|φ(xⱼ)⟩|², fits classical SVM
3. **Evaluation** — reproduces Tables I & II (accuracy, F1, recall, AUC across 4/8/10-qubit configs)

## Quick Start

### 1. Install
```bash
git clone <this-repo>
cd paper_qsvm_fraud_detection

# Create virtualenv
python -m venv .venv && source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt
pip install -e .
```

### 2. Get the dataset
```bash
# Requires Kaggle account — see data/README_data.md
python scripts/download_data.py --output-dir data/raw/
```

### 3. Debug run (fast — ~2 min, 300 samples, 4-qubit)
```bash
python train.py --config configs/config_debug.yaml \
                --csv data/raw/creditcard.csv
```

### 4. Full training (10-qubit + Quantum-SMOTE)
> ⚠️ **Warning:** Computing the full quantum kernel matrix is O(N²) and may take several hours on a CPU. Use `--max-train-samples` to limit training set size.

```bash
python train.py --config configs/config.yaml \
                --csv data/raw/creditcard.csv
```

### 5. Ablation study (Tables I & II)
```bash
python scripts/run_ablation.py --csv data/raw/creditcard.csv \
                                --max-samples 500  # remove for full run
```

## Computational Warning

The quantum kernel matrix is **O(N²)** in training set size. For the full
balanced dataset (~57K samples), this is ~3.2 billion circuit evaluations —
computationally intractable on a CPU simulator.

**Recommendations:**
- Use `--max-samples 500` for initial validation
- Enable `cache_kernel: true` (default) to avoid recomputation
- Use `configs/config_debug.yaml` for smoke testing

The paper likely trained on a downsampled subset; the confusion matrix figures
show ~135K test samples suggesting a much smaller training set.

## High-Risk Implementation Notes

1. **Quantum-SMOTE** (`models/quantum_smote.py`): The rotation synthesis step is the classical interpolation analogue of the paper's quantum rotation. The exact gate sequence from Mohanty et al. 2025 [ref 11] was not available — see `TODO` in source.
2. **SVM C parameter**: Unspecified in paper. Default C=1.0 used. Run `scripts/tune_C.py` to find the best C.
3. **Feature selection**: The exact top-10 KBest features are not enumerated in the paper. Runtime logs will print selected features for comparison against paper Figure 3.
